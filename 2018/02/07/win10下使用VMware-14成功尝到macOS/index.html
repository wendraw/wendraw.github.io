win10 下使用 VMware 14 成功尝到到 macOS

发表于 2018-02-07 | 分类于 技术向 | 阅读次数:

摘要

这一篇文章主要记录 win10 在使用 VMware 时候的一些坑。

闲谈

最近公司安排学习 Android 方面的开发，对于我这个只学过 C#，还只有使用过 winform 以及 WPF 框架的菜鸟来说这确实是一个挑战，但是有挑战就意味着能学到新东西，作为桌面方向的开发者一直对 web 方向总有一种向往之情，或许是因为他们薪水涨的比较快吧（嗯，其实就是这个原因），刚好 C# 群里的大佬听说的在学 Android，立马向我安利了一波 web app，所以我毅然决然的选着了学习一波，但是，我们公司的需求却是 Native app。哈哈哈，有点尴尬，但是这还是挡不住我学习web的心，于是我买来了一本 《React Native 跨平台移动应用开发的书》，开始了我的 web app 之旅。要开发跨平台的 app 第一步自然就是搭建环境咯，并且 iPhone 应用的开发也是需要 macOS，但是我手上只有 Windows 的机器，于是现在就发挥我的程序员精神自己动手丰衣足食（还不是因为我这个穷屌丝买不起 mac 机 -.- ）。

正文

废话不多说，现在回到正文，在 Windows 上搭建 mac 有两种方案，第一，黑苹果——也就是直接将电脑从 Windows 换成 macOS 环境了，但是会存在操作不习惯，固件不兼容等一系列问题，所以我们直接采用第二种。第二，在虚拟机下安装 macOS，这种方法就很好的避免了上一种方法的问题，同时还能自由移植，多处备份。

我们都知道 Windows 自带虚拟机 Hyper-V，但是微软当然不允许自己的产品上可以使用竞争对手的系统了。所以就需要用到今天的「主人公」 VMware，关于 VMware 我就不介绍了，请自行 bing。然后关于 VMware 安装 MacOS 的教程网上一搜都有大把的，所以我也不赘述，推荐几个我参考过的链接。
VMware 12 安装 macOS S 10.12
VMWare14 安装Mac OS系统（图解）

下面我说说安装过程中会遇到的坑：

  1. 安装完成之后启动虚拟机，界面直接跳转到 BIOS 界面；
    这个问题是因为你所下载的 ISO 或 CDR 文件不正确，这里推荐下载 CDR 文件，第一个链接里的百度网盘的链接我下载的 OS X EI 10.11.5 是有效的。然后还有一个注意的在使用 VMware 创建虚拟机是要求使用 ISO 资源文件，所以笔者就想到将自己在官方下的 dmg 通过 UltraISO 工具转成 ISO文件，然后再安装系统，最后的结果就是直接跳到 BIOS 界面，系统不能读取文件。

  2. 在 Windows 下自带的 Hyper-V 与 VMware 冲突
    具体的错误提示如下：
    参考 VMware 官方的解决方法如下 ：
    在启用了 Credential Guard 或 Device Guard 的 Windows 10 主机上运行 Workstation 失败 (2148465)
    ​
    但是不能完全解决问题，重启之后还是会继续开启 Hyper-V 服务，并且在重启的时候提示，配置无法成功，恢复默认配置。
    此时的解决方案是，
    a. 先将 Hyper-V 的服务手动停止，首先进入 任务管理器 –> 选择 服务 –> 点击左下角的 打开服务 –> 找到所有的 Hyper-V 项，将启动类型设置为禁用。

    b. 然后使用管理员启动终端，输入：bcdedit /set hypervisorlaunchtype off。
    如果想开启 Hyper-V，输入：bcdedit /set hypervisorlaunchtype auto。

    至此应该就能成功开启你的 macOS 之旅，合影留念。
